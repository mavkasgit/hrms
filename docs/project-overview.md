# Project Overview

## Описание

**HRMS** — корпоративная система управления кадрами, организационной структурой, отпусками, приказами и документооборотом. Проект спроектирован как монорепозиторий с разделением на backend (FastAPI), frontend (React с Feature-Sliced Design), инфраструктуру (Docker Compose + Nginx) и E2E-тестирование (Playwright). Система интегрирована с OnlyOffice Document Server для генерации, редактирования и просмотра документов.

## Стек технологий

| Слой | Технологии | Версия / Описание |
|---|---|---|
| **Backend** | Python 3.11+, FastAPI, SQLAlchemy (async), Alembic, Uvicorn | Асинхронный бэкенд, структурированный по слоям |
| **Frontend** | React 18, TypeScript, Vite, TailwindCSS | SPA с архитектурой Feature-Sliced Design (FSD) |

| **Infra** | Docker Compose, Nginx | Раздельные профили dev / test / prod, Nginx reverse proxy |
| **E2E** | Playwright (Chromium) | Автоматизированные тесты для UI, API и бизнес-логики |
| **Scripts** | Node.js (`scripts/run.js`), Bash / PowerShell | Кроссплатформенный запуск вспомогательных команд |

## Архитектурные решения

### FastAPI Backend (`backend/app/`)
Используется **слоёная архитектура (Layered Architecture)** с изоляцией ответственности:
1. **HTTP-слой (`api/`)** принимает запросы, валидирует их с помощью Pydantic-схем, вызывает методы сервисов и возвращает ответы.
2. **Бизнес-логика (`services/`)** содержит алгоритмы расчётов, обработку правил (например, генерация отпусков, валидация баланса) и вызовы репозиториев.
3. **Доступ к данным (`repositories/`)** абстрагирует запросы к БД с помощью асинхронного SQLAlchemy API.
4. **Модели (`models/`)** объявляют структуру таблиц БД.

*Правило*: Зависимости направлены сверху вниз: `API -> Services -> Repositories -> DB`. Обратные вызовы строго запрещены.

### React Frontend (`frontend/src/`)
Используется методология **Feature-Sliced Design (FSD)**:
1. **`shared`** — переиспользуемые UI-компоненты (кнопки, инпуты), базовые утилиты, общие хуки, настройки API-клиента.
2. **`entities`** — бизнес-сущности (например, Employee, Vacation, Order). Содержит модель данных, API-методы для этой сущности и простые UI-карточки.
3. **`features`** — интерактивные действия, несущие бизнес-ценность (например, `CreateEmployeeForm`, `RequestVacationButton`).
4. **`pages`** — композиция сущностей и фич в полноценные экраны (например, `EmployeeListPage`, `VacationCalendarPage`).
5. **`app`** — глобальная инициализация (роутер, провайдеры контекста, глобальные стили).

*Правило*: Разрешены импорты только снизу вверх. Срез из `entities` не может импортировать ничего из `features` или `pages`.

### Интеграция с OnlyOffice
OnlyOffice Document Server используется для автоматической сборки документов (приказов, заявлений) по шаблонам и их редактирования прямо в браузере.
- Бэкенд генерирует DOCX/PDF документы, заполняя плейсхолдеры в шаблонах, и сохраняет их.
- OnlyOffice инициализируется на фронтенде через JavaScript API.
- Взаимодействие происходит по схеме:
  1. Фронтенд запрашивает параметры редактора у бэкенда.
  2. Бэкенд возвращает конфигурационный JSON (включая JWT-токен для авторизации в OnlyOffice).
  3. Фронтенд рендерит фрейм OnlyOffice, передавая ему `ONLYOFFICE_PUBLIC_URL` и адрес документа.
  4. OnlyOffice скачивает документ с бэкенда, используя внутренние адреса, и отправляет изменения обратно через callback (`BACKEND_INTERNAL_CALLBACK_URL`).

## Вспомогательные скрипты (`scripts/`)
Для обеспечения кроссплатформенности (Windows/Linux/macOS) запуск большинства команд обернут в `scripts/run.js`:
- Он автоматически определяет ОС (`win32` или Unix).
- Запускает соответствующий скрипт: `.js` (Node.js), `.ps1` (PowerShell) или `.sh` (Bash/WSL).
- Позволяет разработчикам использовать одинаковый интерфейс команд независимо от рабочей станции.

Основные утилиты:
- `wait-for-postgres` — опрашивает БД, блокируя выполнение следующих команд, пока СУБД не будет готова принимать соединения.
- `run-migrate` — запускает миграции Alembic (`alembic upgrade head`) только после того, как БД станет доступной.
- `run-backend` — запускает локальный сервер Uvicorn с автоперезапуском при изменении кода.
