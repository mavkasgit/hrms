# Deployment HRMS

## Обзор

Деплой приложения **HRMS** осуществляется с использованием **Docker Compose**. Стек включает в себя бэкенд на FastAPI, фронтенд на React, базу данных PostgreSQL и Document Server OnlyOffice для работы с документами. Обратный прокси (Nginx) обеспечивает маршрутизацию трафика и поддержку SSL.

## Сетевые режимы работы (Порты)

Проект поддерживает два параллельных режима работы с портами и сетью:

1. **Режим единой общей сети (через порт 80 / 443)**:
   - Используется внешняя сеть `shared_gateway`.
   - Доступ ко всем проектам (`hrms`, `ktm2000`, `ekranchik`) осуществляется через единую точку входа на порту 80/443 (с использованием центрального Nginx / Traefik reverse proxy и доменных имен).
   - Локальный Nginx контейнер подключается к сети `shared_gateway`.

2. **Автономный режим (прямой доступ)**:
   - Доступ к HRMS осуществляется напрямую через порт хоста **8081** (проброшен на порт 80 контейнера Nginx в `docker-compose.prod.yml`).
   - Для сравнения: `ktm2000` доступен на порту **8082**, а `ekranchik` на порту **8083**.

## Окружения

Проект поддерживает три основных окружения:
1. **dev**: База данных PostgreSQL и OnlyOffice работают в Docker, бэкенд и фронтенд запускаются локально.
2. **test**: Полный стек запускается в контейнерах. Порт входа (через Nginx): `8080`.
3. **prod**: Полный стек запускается в контейнерах. Порт входа (через Nginx): `8081` (в автономном режиме) или `80/443` (через `shared_gateway`).

## Сборка и деплой

### Сборка и запуск Production окружения
```bash
# 1. Загрузите файлы проекта на сервер
# 2. Перейдите в корень проекта
cd hrms

# 3. Скопируйте env-шаблон и заполните его реальными данными
cp .env.example .env.prod

# 4. Запустите стек через Docker Compose
npm run docker:prod:up
```

### Остановка стека
```bash
npm run docker:prod:down
```

## Контракт переменных окружения

Специфические переменные для HRMS (настраиваются в `.env.prod` / `.env.dev`):
- `DATABASE_URL`: Строка подключения к PostgreSQL (например, `postgresql+asyncpg://user:pass@localhost:5432/db`).
- `ONLYOFFICE_PUBLIC_URL`: Публичный URL Document Server для браузеров (обычно `http://<domain>/onlyoffice` или `http://localhost:8085`).
- `ONLYOFFICE_INTERNAL_URL`: Внутренний URL для общения бэкенда с OnlyOffice (`http://nginx/onlyoffice` или `http://localhost:8085`).
- `BACKEND_INTERNAL_CALLBACK_URL`: URL, используемый OnlyOffice для обратных вызовов на бэкенд (`http://backend:8000`).

## Чек-лист деплоя

- [ ] Файл `.env.prod` создан и заполнен корректными учетными данными.
- [ ] Порты 8081 или 80/443 открыты в UFW.
- [ ] Контейнеры запущены: `docker ps` показывает статус `Up (healthy)`.
- [ ] OnlyOffice доступен и JWT авторизация работает.
- [ ] Бэкапы баз данных настроены на регулярный запуск.
