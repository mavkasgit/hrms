# Agents Summary HRMS

Сводка для субагентов и ИИ-ассистентов: ключевые правила разработки и архитектурный контекст.

## Реестр портов и сетевые режимы

Приложение может работать в двух параллельных режимах:
1. **Сетевой (shared_gateway)**: Маршрутизация через единую сеть на порту **80/443**.
2. **Автономный (прямой доступ)**: Доступ по выделенным портам для каждого проекта.

| Порт | Назначение / Окружение | Автономный доступ | Сетевой доступ |
|---|---|---|---|
| **5173** | Frontend (Vite dev server) | Локально в dev | — |
| **8000** | Backend API (FastAPI) | Локально в dev | — |
| **8085** | OnlyOffice Document Server | dev | — |
| **8080** | Nginx entrypoint (test) | `http://localhost:8080` | — |
| **8081** | Nginx entrypoint (prod, HRMS) | `http://localhost:8081` | `http://<domain>/` (через порт 80/443) |
| **8082** | Nginx entrypoint (prod, KTM-2000) | `http://localhost:8082` | `http://<domain>/` (через порт 80/443) |
| **8083** | Nginx entrypoint (prod, Экранчик) | `http://localhost:8083` | `http://<domain>/` (через порт 80/443) |
| **9222** | Chrome CDP (отладка) | dev (тестирование) | — |

## Основные субагенты

- `@browser-checker`: Используется для всех проверок страниц в реальном браузере (навигация, скриншоты, консоль, сеть).
- `@docs-maintainer`: Запускается для актуализации документов в `docs/` после изменений в коде.
- `@test-runner`: Запуск регрессионных или точечных E2E тестов.
- `@server-operator`: Запуск команд на удаленном хосте через MCP `ssh-connect`.
- `@sftp-operator`: Загрузка и скачивание релизов/логов через MCP `sftp-connect`.

## Ключевые правила разработки

1. **Frontend (React)**:
   - Строгое соблюдение методологии **Feature-Sliced Design (FSD)**. Запрещены перекрестные импорты между фичами или импорт снизу-вверх.
   - Компоненты стилизуются через TailwindCSS. Использование Radix UI и Lucide icons приветствуется.
2. **Backend (FastAPI)**:
   - Использование асинхронных эндпоинтов и слоеной архитектуры (`API -> Service -> Repository`).
   - Валидация входных данных через Pydantic-схемы.
3. **E2E-тесты**:
   - Использование **Page Object Model** (POM) в `e2e/pages/`.
   - Импорт `{ test, expect }` напрямую из `@playwright/test` или из `e2e/fixtures.ts` для поддержки CDP.
